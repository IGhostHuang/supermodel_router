## 1. 背景

**老大原话** (2026-06-17 22:00):
> "smr 每次切换模型的时候, 也同步上下文和任务给新的模型, 不然对话连不上. 同时要判断回传的信息是否过期, 已过期的话, 加上过期标记."

**核心痛点**:
- chain rotation 切到新 candidate 时, 新模型只看到原始 prompt, 不知道前面模型为什么失败 / 部分响应, 容易"对话断层"
- 如果请求耗时 > 30 min, 新模型还在用"最新"的口吻回答过时的事实, 缺乏过期提示

**目标**: SMR v3.4.0 实现 3 大机制:
1. 非流式切换 → 注入"上下文桥接"system prompt
2. 流式切换 → 切到新 candidate 第一个 chunk 前发 `_smr_bridge` SSE sentinel
3. 全场景 → 整个请求耗时 > 阈值时标 `stale=true` (默认 1800s/30 min)

## 2. 架构改动

### 2.1 新增模块: `supermodel_router/context_bridge.py` (285 行)

核心类:
- **`ContextBridge`** — 单例, 持有 enabled / threshold / max_history / sentinel_enabled / inject_template
- **`SwitchRecord`** — 单次切换记录: `from_provider / from_model / from_full_path / status / http_code / error / partial_text / timestamp`
- **`SwitchHistory`** — deque(maxlen=5), 自动滚动
- **`DEFAULT_INJECT_TEMPLATE`** — 默认 inject 模板
- **`build_sse_sentinel(records, stale, age, threshold)`** — 生成 `{"_smr_bridge": {...}}` JSON
- **`is_stale(age_seconds)` / `age_seconds(start_time)` / `get_stats()`** — 状态查询

**关键设计**:
- is_stale 公式: 用整个请求耗时 `time.time() - request_start_time`, **不是** `switch_time - request_start_time` (后者在 1 次切换场景下永远 ~0, 永远不 stale)
- 历史滚动: deque(maxlen=max_history), 自动淘汰最老
- 模板渲染: 用 `str.format()` 替换 `{version} / {n_attempts} / {attempt_blocks} / {age_minutes}`, try/except 兜底失败时返回原模板

### 2.2 修改: `supermodel_router/openai_routes.py` (重写 445 行)

- **`init()`** 三参: `(app_registry, app_engine, app_bridge)` 把 ContextBridge 单例注入
- **非流式切换**: 切到新 candidate 前, 调 `bridge.inject_into_messages(messages, history)`, 把 system prompt 插入到 messages[0]
- **非流式响应 `_router`**: 加 `switched_from: list[dict] / stale: bool / age_seconds: int / stale_threshold_seconds: int` 4 字段
- **流式 `_stream_generator`**: 切到新 candidate 时, 第一个 chunk 前 yield `data: {"_smr_bridge": {...}}\n\n` SSE sentinel
- **LSP 假报解**: `Any` type hint, 跟 v3.3 同型处理

### 2.3 修改: `supermodel_router/app.py`

- `from .context_bridge import ContextBridge`
- `context_bridge: ContextBridge | None = None` 全局
- lifespan 中 `ContextBridge(config)` 实例化
- `openai_init(registry, engine, context_bridge)` 三参注入

### 2.4 修改: `supermodel_router/admin_api.py`

- 加 `init()` 第 5 参 `app_bridge: ContextBridge | None = None`
- 加 3 个 endpoint:
  - `GET  /v1/admin/context_bridge` → config + stats
  - `PUT  /v1/admin/context_bridge` → 热更新 enabled/threshold/max_history/sentinel_enabled
  - `POST /v1/admin/context_bridge/reset` → 清零 stats

### 2.5 修改: `supermodel_router/version.py`

- VERSION 3.3.0 → 3.4.0
- BUILD_DATE 2026-06-17
- changelog 段加 v3.4.0 记录

### 2.6 修改: `config.yaml`

加 `context_bridge` 段 (默认开启, 30 min 阈值):

```yaml

  enabled: true
  stale_threshold_seconds: 1800
  max_history: 5

```

### 2.7 新增: `tests/test_context_bridge_e2e.py` (430 行)

3 场景:
1. **非流式切换**: mock_a 401 → mock_b 200, 验证 `_router.switched_from` 1 条 + stale=False
2. **流式切换**: mock_a 流式 401 → mock_b 流式 200, 验证 SSE `_smr_bridge` sentinel + version=3.4.0
3. **过期标记**: mock_a 自身 sleep 3s 再 401, threshold=2s, 验证 `_router.stale=True + age_seconds=3`

**E2E 关键 trick**:
- 用 `run.py --port 16473` 启 SMR (不用 `__main__.py`, 后者只改 app_mod.config, 子模块用 `from .config import config` 拿原实例)
- 启动前必 rm 3 个持久化 state 文件 (`penalty_state.json / engine_stats.json / model_rules_state.json`), 否则上轮 mock_a 失败 penalty=0.4 让 B 排前不切链
- 进程清理用 glob `/proc/[0-9]*/cmdline` 找 `run.py.*16473` 杀, `fuser` 不可靠

### 2.8 升级: `run_smr_pyinstaller.py` (PyInstaller 入口)

加 argparse: `--config / --host / --port / --log-level`, 跟 `run.py` 行为对齐 (之前只读 config.yaml 不接命令行, 导致 binary 启动后只能用默认 6473).

### 2.9 升级: `smr.spec` (PyInstaller 打包配置)

hiddenimports 加 7 个新模块: `admin_api / admin_ui / context_bridge / model_manager / model_rules / openai_routes` (原来只列了 app / classifier / cli / config / detector / engine / models / version).

## 3. 自验结果

### 3.1 E2E 测试 ✅ 12/12 PASS

```
[场景 1] 非流式切换 — A 401 → B 200
  HTTP 200, _router.chain_position: 1
  _router.switched_from: 1 entries [mock_a/mock-model-a status=http_401 http=401 stale=False]
  _router.stale: False, _router.age_seconds: 0

[场景 2] 流式切换 — A 流式 401 → B 流式 200 + sentinel
  收到 5 行 SSE
  sentinel found: True (_smr_bridge.version=3.4.0, switched_from_count=1, stale=False, age_seconds=3)

[场景 3] 过期标记 — threshold=2s, mock_a 自身 sleep 3s 再 401
  HTTP 200, _router.stale: True, _router.age_seconds: 3
  stale_threshold_seconds: 2
```

**关键 bug 修复** (调试中踩):
1. ❌ SMR 启动用 `__main__.py` → 改 `run.py` (直接 `cfg.load()` 替换全局 config 实例)
2. ❌ E2E 残留进程占用 16473 → `fuser -k` 不工作, 改 glob `/proc/*/cmdline` 杀
3. ❌ `chain[0]` 是 B 不是 A → `penalty_state.json` 持久化上轮 mock_a penalty=0.4, E2E 启动前必 rm
4. ❌ `is_stale` 永远 False → 公式错 (`switch_time - request_start_time` 永远 ~0), 改 `time.time() - request_start_time` 整个请求耗时
5. ❌ 场景 3 client sleep 3s 不算 → 改 mock_a 自身 `time.sleep(3)` 再返 401, SMR 端 request 实际耗时 3s

## 5. 关键决策沉淀

### 5.1 is_stale 公式

```python
# ✅ 对: 整个请求耗时
age = time.time() - request_start_time
stale = age > threshold

# ❌ 错: 切换耗时 (1 次切换场景下永远 ~0)
age = switch_time - request_start_time
stale = age > threshold
```

### 5.2 PyInstaller 启动入口

```python
# ✅ 对: run.py 走 `if __name__ == "__main__"` 有 argparse, 接 --port
# ❌ 错: run_smr_pyinstaller.py 没 argparse 时, binary 启动后只能用 config.yaml 默认值
```

### 5.3 持久化 state 处理

```bash
# ✅ 对: E2E 启动前必 rm
rm -f penalty_state.json engine_stats.json model_rules_state.json

# ❌ 错: 不 rm, 上轮 mock_a 失败 penalty=0.4 让 B 排前不切链
```

### 5.4 LSP 假报

```python
# ✅ 对: 用 Any type hint 解
engine: Any = None
registry: Any = None
# ❌ 错: LSP 报 "engine is None" 时改代码逻辑, 实际是静态分析看不到 init() 时填充
```

### 5.5 进程清理

```python
# ✅ 对: 扫 /proc 找 cmdline 匹配
for pid_dir in glob.glob("/proc/[0-9]*"):
    try:
        with open(f"{pid_dir}/cmdline") as f:
            if b"run.py" in f.read() and b"16473" in f.read():
                os.kill(int(pid_dir.split("/")[-1]), 9)
    except: pass

# ❌ 错: fuser -k (WSL 下不可靠)
```

## 7. 后续可选优化 (非阻塞)

1. **Admin UI 加 context_bridge 面板** — 当前只有 API, 没 UI 按钮, 可在 `admin_ui.py` 加 toggle (低优先级)
2. **stale 触发 UI 提示** — 客户端解析 `_router.stale=true` 后弹"信息可能已过期"提示 (需 SMR 客户端配合)
3. **partial_text 真正截取** — 当前流式切换时只保留 `from_full_path` 标记, 没真把已输出的文本拼回新 candidate context (SSE chunk 内容截取实现复杂, 留待 v3.5)
4. **sentinel 兼容老客户端** — 严格 SSE 协议下, 老客户端解析 `data: {"_smr_bridge":...}` 可能 panic, 可加 feature flag `sentinel_legacy_mode` 改用注释行 `:_smr_bridge{...}` 兼容
5. **PyInstaller 减小体积** — 当前 23 MB, 可加 `excludes=['numpy', 'pandas', ...]` 进一步压 (已加了, 仍 23M 是因为 httpx + uvicorn 占大头)
