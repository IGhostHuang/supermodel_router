## 🎯 问题 (Why)

### 当前 SMR 切换模型的盲区

```
┌─────────────────────────────────────────────────────────────────────────────┐
│ 客户端 (mainbot / OpenAI SDK)                                                │
│   发出 messages=[u1, a1, u2, a2, u3] (全量, 客户端维护 history)               │
│                              ↓                                              │
│ SMR chain = [model A, model B, model C] (按评分排序)                          │
│                              ↓                                              │
│   model A 试 → 401 (失败)                                                    │
│   切到 model B → B 看到全量 messages → 生成完整回复                            │
│                              ↓                                              │
│   客户端收到: "B 的完整回复"                                                   │
│   客户端 history: assistant = B 的回复 → 后续 user 消息接着它                  │
└─────────────────────────────────────────────────────────────────────────────┘
```

**隐藏问题**:
1. **流式场景**: SMR 当前实现是失败时**直接终止** (`yield error + break`), **不切链**; 老大的"切换时同步上下文" 在流式上**根本没有生效**
2. **对话断层**: 假设 model A 流式输出了 50 tokens, 然后 timeout, SMR 切到 model B — 客户端只看到 B 的回复,**不知道 A 已经说了一半**; 用户感觉回答莫名其妙
3. **过期回传**: 假设 model A 5xx, SMR 切到 B 成功 — 但 B 不知道"A 刚才超时了,我应该重新给个完整答案" — 直接基于原始 messages 回答,**前面对话用户可能觉得过时**
4. **跨 30min 跨天**: 假设一个长 session,客户端发了一个 1 小时前的 messages (某些 client 缓存,某些 retry chain),新模型基于 1 小时前的对话回复 — 完全可能过期

---

## 🏗️ 设计 (What)

### 3 大机制

#### 机制 1: SwitchHistory 跟踪 (per-request)

每次切 candidate, 记录到 in-memory 列表 (随 request 生命周期, 不持久化):

```python
@dataclass
class SwitchRecord:
    from_provider: str           # 切出的 provider
    from_model: str              # 切出的 model
    from_full_path: str          # "openrouter/gpt-oss-120b:free"
    partial_text: str            # 已经流出的 chunk 拼接 (流式) / "" (非流式)
    switch_time: float           # 切换时间戳 (epoch)
    response_status: str         # "timeout" / "http_5xx" / "http_401" / "stream_error"
    http_code: int               # 0 / 401 / 500 ...
    error_message: str           # 详细错误
    request_start_time: float    # 整个请求的起始时间 (用于算 age_seconds)
    attempt_index: int           # 第几次尝试 (0=first)
```

**保留最近 5 次切换** (configurable) — 多次重试都失败时, 新 model 看到完整失败链。

#### 机制 2: ContextBridge.inject 注入

切到下一个 candidate 前, 改写新 request 的 `body.messages`, 在头部插入 system prompt:

```python
SYSTEM_TEMPLATE = """[SMR 上下文桥接 v3.4.0]

你正在接续一个多模型对话. 前面有 {N} 次模型尝试:

{ATTEMPT_BLOCKS}

你的任务: 
1. 直接基于**用户最后一条消息**和**已有对话历史**给一个完整回答
2. 不要重复前面模型已经成功输出的内容
3. 如果前面有部分响应 (PARTIAL), 把它视为"另一个 AI 的草稿", 你可以在此基础上完善, 也可以完全重写
4. 如果切到你的时间已经超过 {AGE_MINUTES} 分钟, 请明确提醒用户信息可能过期

[SMR 桥接结束 — 下面是真实对话]
"""
```

**插入位置**: `messages[0]` 如果是 system, 在它**前面**插一个 system; 否则在 messages 列表**最前面**插一个 `{role: "system", content: SYSTEM_TEMPLATE}`

**client 透明**: 这是 server-side 注入, client 不知道, 跟 OpenAI 协议完全兼容 (只是 server 在原 messages 基础上多加了 1 个 system message)

#### 机制 3: Staleness 过期标记

```python
def is_stale(switch: SwitchRecord, threshold_s: int = 1800) -> bool:
    return (switch.switch_time - switch.request_start_time) > threshold_s
```

**2 个标记出口**:

1. **流式响应**: 切到新 candidate 的**第一个 SSE chunk 前**插入一个 sentinel:
   ```
   data: {"_smr_bridge": {"switched_from": [...], "stale": true, "age_seconds": 1850, "version": "3.4.0"}}\n\n
   ```
   客户端可以读这个 JSON 块识别切换 + 过期。

2. **非流式响应**: 在 `result._router` 加字段:
   ```python
   result["_router"]["switched_from"] = [...list of SwitchRecord dicts...]
   result["_router"]["stale"] = True
   result["_router"]["age_seconds"] = 1850
   ```

## 📐 实施 (How)

### 边界 (4 编码原则)
- ✅ **in**: SMR 内部切换时 inject system prompt + 写 metadata; 跟踪 per-request switch history
- ❌ **out**: 不改 OpenAI 协议; 不持久化 switch history (in-memory only); 不强制 client 读 sentinel
- 💰 **成本**: 0 额外外部调用; < 1KB 内存 per request; 启动时多 1 个模块
- ⚠️ **异常**: inject 失败 → fallback 到原 messages (不阻断); stale 计算错误 → 默认为 False (不误标)
- 🔍 **可观测性**: `engine_stats.json` 加 `bridge_injections_total` + `bridge_stale_total` 2 个计数; admin UI 加"上下文桥接"面板

### 关键代码骨架

```python
# context_bridge.py
@dataclass
class SwitchRecord:
    from_provider: str
    from_model: str
    from_full_path: str
    partial_text: str
    switch_time: float
    request_start_time: float
    response_status: str
    http_code: int
    error_message: str
    attempt_index: int
    
    def is_stale(self, threshold_s: int) -> bool:
        return (self.switch_time - self.request_start_time) > threshold_s
    
    def age_seconds(self) -> int:
        return int(self.switch_time - self.request_start_time)


class ContextBridge:
    """v3.4.0: 上下文桥接 + 过期标记"""
    
    def __init__(self, cfg: dict):
        self.enabled = cfg.get("enabled", True)
        self.stale_threshold_s = cfg.get("stale_threshold_seconds", 1800)
        self.max_history = cfg.get("max_history", 5)
        self.inject_template = cfg.get("inject_template", DEFAULT_TEMPLATE)
        self.sentinel_enabled = cfg.get("sentinel_enabled", True)
        # stats
        self._stats = {
            "injections_total": 0,
            "stale_marks_total": 0,
            "switch_records_total": 0,
        }
    
    def build_inject_message(self, history: list[SwitchRecord]) -> dict | None:
        """根据 history 构造要插入的 system message"""
        if not self.enabled or not history:
            return None
        # ... (按模板生成 content)
        return {"role": "system", "content": content}
    
    def inject_into_body(self, body: dict, history: list[SwitchRecord]) -> dict:
        """把 system message 插入 body.messages 头部, 复制 body 避免 mutate"""
        msg = self.build_inject_message(history)
        if not msg:
            return body
        new_body = {**body, "messages": [msg] + list(body.get("messages", []))}
        self._stats["injections_total"] += 1
        return new_body
    
    def build_sse_sentinel(self, history: list[SwitchRecord]) -> str:
        """构造流式切换 sentinel (SSE 格式)"""
        if not self.sentinel_enabled or not history:
            return ""
        last = history[-1]
        payload = {
            "_smr_bridge": {
                "version": "3.4.0",
                "switched_from": [asdict(h) for h in history],
                "stale": last.is_stale(self.stale_threshold_s),
                "age_seconds": last.age_seconds(),
            }
        }
        return f"data: {json.dumps(payload, ensure_ascii=False)}\n\n"
    
    def record_switch(self, record: SwitchRecord):
        self._stats["switch_records_total"] += 1
        if record.is_stale(self.stale_threshold_s):
            self._stats["stale_marks_total"] += 1
    
    def get_stats(self) -> dict:
        return dict(self._stats)
    
    def reset_stats(self):
        for k in self._stats: self._stats[k] = 0
```

| 1 | **非流式正常切换** (model A 401 → B 成功) | response._router.switched_from 含 A 的 SwitchRecord, stale=false, B 收到 inject system |

## 📊 影响范围 + 兼容性

- ✅ **OpenAI 协议兼容**: 多加 1 个 system message 合法; SSE sentinel 是 `data: {...}` 标准格式
- ✅ **现有 client 不感知**: 不知道 server 内部切了几次, 看到的还是 1 个完整 response 或 1 个完整 stream
- ✅ **memory + 性能**: < 1KB per request, in-memory, request 结束即 GC
- ✅ **fail-safe**: inject 失败 → fallback 到原 body (SMR 行为完全跟 v3.3 一样)
- ⚠️ **sentinel 兼容性**: 严格 OpenAI SDK 解析 `data: {...}` 时如果看到 `_smr_bridge` key 可能会 warn, 不会 fail (主流 SDK 都 forward raw chunk)
